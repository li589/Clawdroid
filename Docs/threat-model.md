# Clawdroid 威胁模型说明书

## 1. 文档目的

本文档用于定义 `Clawdroid App` 与 `Clawdroid Magisk 模块` 在设计、实现、测试和上线前必须共同遵守的安全边界与威胁假设。本文档的目标不是证明系统“绝对安全”，而是明确：

- 系统要保护什么
- 系统信任谁、不信任谁
- 主要攻击面在哪里
- 每类威胁通过什么机制缓解
- 哪些风险被接受、哪些风险必须阻断

本文档与 [基础方案设计.md](file:///d:/temp_desktop/Proj/Clawdroid/Docs/%E5%9F%BA%E7%A1%80%E6%96%B9%E6%A1%88%E8%AE%BE%E8%AE%A1.md) 和 [protocol.md](file:///d:/temp_desktop/Proj/Clawdroid/Docs/protocol.md) 配套使用，是进入编码、联调和安全审查前的正式依据。

## 2. 适用范围

本文档覆盖以下对象：

- `Clawdroid App` 主进程
- `Clawdroid App` 内的 `AccessibilityService`
- `Clawdroid App` 内的 LSPosed 适配代码
- `Clawdroid Magisk 模块`
- `ClawRuntime`
- App 与 ClawRuntime 之间的本地 IPC 协议
- Root 侧的配置、审计、日志与缓存目录

本文档不覆盖以下对象：

- 云端模型服务本身的内部安全设计
- 目标第三方 App 的内部业务逻辑安全
- Android 内核级 0day 与 Bootloader 级攻击
- 用户主动关闭安全限制后的自破坏行为

## 3. 系统概览

Clawdroid 只有两个部署单元：

1. **Clawdroid App**
   - 承担 UI、任务编排、模型交互、无障碍、LSPosed 模块代码和 IPC 客户端。

2. **Clawdroid Magisk 模块**
   - 承担 Root 守护进程、启动脚本、配置、审计与高权限执行能力。

运行时存在三个代码位置：

1. **App 主进程**
2. **Root 守护进程**
3. **目标 App 进程中的 LSPosed 注入代码**

安全设计要求将“代码运行位置”和“部署单元”区分开。LSPosed 注入代码属于同一个 APK 的一部分，但其运行位置跨越到了第三方进程，因此必须单独建模其信任边界。

## 4. 安全目标

Clawdroid 的核心安全目标如下：

1. **限制 Root 面暴露**
   - Root 守护进程不向公网或本地 TCP 暴露开放接口，只接受受控本地 IPC。

2. **限制调用方身份**
   - 只有受信任的 `Clawdroid App` 才能调用 ClawRuntime 的高权限能力。

3. **限制高危动作范围**
   - 输入注入、Shell 执行、文件读取和截图都必须受能力、参数和路径约束。

4. **限制适配代码影响面**
   - LSPosed 适配代码只能在明确白名单目标上工作，不能扩大到系统全局。

5. **保证可审计性**
   - 每个高危请求都必须可追踪到发起方、参数摘要、执行结果和时间点。

6. **保证降级可用**
   - 当 Root、SELinux 修补或 LSPosed 能力不可用时，系统应降级，而非错误暴露更多权限。

## 5. 非目标

以下目标明确不在本系统承诺范围内：

- 防御设备所有者以 Root 身份对系统做任意破坏
- 防御 Bootloader 解锁后更底层的物理攻击
- 保护已经被系统级恶意代码完全接管的设备
- 将 Clawdroid 作为通用提权框架对外开放
- 保证所有 ROM、所有 Android 版本、所有 Root 方案上的完全等价行为

## 6. 资产清单

以下资产需要重点保护：

### 6.1 高价值资产

- `ClawRuntime` 的 Root 执行能力
- IPC 通道及其会话状态
- 能力令牌、挑战信息、会话信息
- 输入注入能力
- 受限 Shell 执行能力
- 受限文件读取能力
- 截图能力

### 6.2 敏感数据资产

- 截图内容
- 目标 App 页面结构和上下文
- 审计日志
- 运行日志
- 配置文件
- 允许包名和签名白名单

### 6.3 完整性资产

- `protocol.md` 中冻结的协议语义
- 错误码语义
- Magisk 模块中的启动脚本
- ClawRuntime 的配置文件
- App 内的目标应用适配元数据

## 7. 信任模型

### 7.1 可信组件

在默认威胁模型下，以下组件被视为受信任但仍需最小授权：

- 已安装且签名匹配的 `Clawdroid App`
- 已安装且来源可信的 `Clawdroid Magisk 模块`
- `ClawRuntime`
- 协议中定义的固定动作集

### 7.2 条件可信组件

以下组件只在特定条件下被部分信任：

- LSPosed 适配代码
  - 仅在目标包命中白名单、版本命中范围且适配模块未熔断时视为可用
- `AccessibilityService`
  - 仅在系统确认授权后视为可用
- `MediaProjection`
  - 仅在授权有效期内视为可用

### 7.3 默认不可信对象

以下对象默认不可信：

- 同设备上的其他普通 App
- 与 `Clawdroid App` 处于同一 SELinux 域的无关 App
- 任意未签名或签名不匹配的 APK
- 任意尝试直接连接 IPC 的本地进程
- 任意未在白名单中的目标应用
- 来自模型输出但未经过规划器和参数校验的原始执行指令

## 8. 攻击者模型

Clawdroid 主要面对以下攻击者：

### 8.1 本地恶意 App

能力：

- 运行于普通应用沙盒
- 可能与 `Clawdroid App` 处于相同 SELinux 域
- 尝试探测、连接或伪造本地 IPC
- 尝试诱导系统暴露额外能力

目标：

- 盗用 Root 守护进程能力
- 读取敏感文件或截图
- 执行输入注入或 Shell 命令

### 8.2 被污染的目标 App 进程

能力：

- 目标 App 自身已被热修复、广告 SDK 或其他注入逻辑污染
- 与 LSPosed 适配代码运行在同一进程
- 可能尝试读取适配代码的上下文数据或影响其行为

目标：

- 干扰适配结果
- 伪造页面状态
- 诱导执行错误点击或错误操作

### 8.3 协议滥用者

能力：

- 拥有合法 App 调用入口
- 反复调用高危动作
- 发送过大载荷、超时请求、非法参数或重复请求

目标：

- 导致守护进程资源耗尽
- 造成重复点击、重复执行、重复截图
- 通过高频调用突破预期控制

### 8.4 具备更高权限的本地攻击者

能力：

- 已取得 Root 或能修改 Magisk 模块文件
- 可读写 ClawRuntime 配置、日志和缓存目录

目标：

- 篡改配置放大能力
- 替换二进制或脚本
- 删除审计日志

说明：

这类攻击者超出了普通应用级威胁边界，系统无法完全防御，但仍应通过完整性校验、最小暴露和审计降低风险。

## 9. 信任边界

Clawdroid 至少跨越以下关键边界：

1. **App 沙盒边界**
   - `Clawdroid App` 与其他普通 App 之间

2. **App 到 Root 守护进程边界**
   - 这是整个系统最关键的权限提升边界

3. **App 主进程到目标 App 进程边界**
   - 通过 LSPosed 适配代码跨进程进入第三方 App

4. **协议边界**
   - 请求帧、响应帧、事件帧必须被视为不可信输入并进行严格解析

5. **文件系统边界**
   - 配置、缓存、审计、日志等目录均需要独立保护

6. **云端模型输出边界**
   - 模型输出不是可信执行指令，只能作为规划输入

## 10. 攻击面清单

### 10.1 IPC 攻击面

- 未授权本地进程尝试连接 UDS
- 伪造请求帧或发送超大负载
- 重放历史请求
- 构造未知动作或非法能力字段

### 10.2 认证与会话攻击面

- 伪造包名
- 利用同 UID 或同域混淆调用方
- 复用旧的挑战令牌或过期会话
- 劫持未完成认证的连接

### 10.3 执行动作攻击面

- 输入注入到错误坐标
- 受限 Shell 退化为任意命令执行
- 文件读取越过白名单路径
- 截图结果包含敏感数据并被持久化

### 10.4 适配代码攻击面

- 目标 App 升级后结构变化导致误识别
- 非白名单进程被错误注入
- 注入代码被目标进程内其他代码干扰

### 10.5 配置与日志攻击面

- 白名单配置被篡改
- 审计日志被删除或伪造
- 调试日志泄露敏感内容
- 缓存截图长期滞留

### 10.6 可用性攻击面

- 高频请求导致守护进程资源耗尽
- 长时间截图或 Shell 执行阻塞队列
- 订阅事件过多导致内存压力

## 11. 主要威胁与缓解措施

以下威胁按 STRIDE 风格进行归类，但采用更贴近实现的描述。

### 11.1 身份伪造

**威胁**

- 恶意 App 伪装成 `Clawdroid App` 连接 ClawRuntime
- 进程伪造包名或试图利用同域权限混淆身份

**缓解**

- 使用对端凭证校验 PID 和 UID
- 校验包名与签名摘要
- 首次连接采用挑战应答
- 使用会话 TTL 和能力令牌
- 未通过认证前不允许进入 `Ready`

**剩余风险**

- 若设备已被更高权限恶意代码接管，则普通身份校验机制可能失效

### 11.2 权限滥用

**威胁**

- 合法调用方调用超出授权范围的能力
- 低风险会话尝试调用高危动作

**缓解**

- 每个动作都绑定 `capability`
- 会话级能力画像决定动作是否开放
- 高危动作必须额外校验能力令牌
- 默认拒绝未授权动作，返回 `ERR_CAPABILITY_NOT_GRANTED`

### 11.3 参数注入与协议滥用

**威胁**

- 发送非法参数、超长字符串、越界坐标、超大文件读取范围
- 通过未知动作探测内部实现

**缓解**

- 严格执行 `protocol.md` 中的参数结构和范围校验
- 限制请求负载大小
- 未知动作统一返回 `ERR_ACTION_NOT_ALLOWED`
- 失败请求也写入审计或速率统计

### 11.4 重放攻击

**威胁**

- 复用历史 `request_id`
- 复用已抓取的挑战响应或旧会话
- 导致重复点击、重复截图、重复命令执行

**缓解**

- `request_id` 去重与短期缓存
- 挑战令牌设置 TTL
- 会话过期后必须重新认证
- 高危动作结合时间戳与会话上下文检查

### 11.5 输入注入误用

**威胁**

- 模型输出错误导致点击敏感位置
- 页面结构变化导致适配坐标失真

**缓解**

- 模型输出必须经过规划器和页面验证
- 高危点击前优先通过 UI 语义确认
- 坐标注入限制在当前显示区域内
- 失败达到阈值后进入适配熔断

**剩余风险**

- 自动化系统始终存在误点击残余风险，需通过审计和逐步确认降低影响

### 11.6 Shell 扩权

**威胁**

- 受限 Shell 退化为任意命令执行
- 利用命令模板中的变量拼接注入恶意参数

**缓解**

- `exec_shell_limited` 只允许命中白名单模板
- 参数采用结构化字段，不直接拼接任意字符串
- 限制超时、输出大小和执行频率
- Shell 执行默认高审计级别

### 11.7 文件读取越权

**威胁**

- 读取任意路径
- 通过路径穿越或软链接绕过白名单

**缓解**

- 路径必须命中白名单前缀
- 解析规范化路径后再做校验
- 禁止相对路径与路径穿越片段
- 设置读取大小上限

### 11.8 截图与隐私泄露

**威胁**

- 截图中包含隐私信息
- 截图被日志或缓存长期保存

**缓解**

- 截图结果默认只用于瞬时任务上下文
- 审计中只记录摘要，不记录原图
- 缓存目录设置 TTL 和定期清理
- 调试日志中不直接打印截图内容

### 11.9 适配代码失效

**威胁**

- 目标 App 升级后页面结构变化
- 适配逻辑误判页面状态

**缓解**

- 每个目标应用维护版本适配范围
- 非兼容版本返回明确适配错误码
- 连续失败触发熔断
- 回退到无障碍或人工确认路径

### 11.10 审计缺失与抵赖

**威胁**

- 高危动作无法追踪
- 调用方事后否认执行过动作

**缓解**

- 每个高危请求记录 `request_id`、`session_id`、`caller_uid`、`action`、`arg_digest`、`result_code`
- 审计日志与运行日志分离
- 日志按时间滚动并限制权限

### 11.11 拒绝服务

**威胁**

- 高频 `subscribe_events` 或 `capture_screen` 导致资源耗尽
- 重复高危动作阻塞正常任务

**缓解**

- 会话限流
- 动作级并发控制
- 订阅数量上限
- 输出和负载大小限制
- 守护进程健康检查与降级策略

### 11.12 配置篡改

**威胁**

- 修改白名单、能力开关、日志级别或路径范围
- 通过配置开启过度权限

**缓解**

- 配置文件权限最小化
- 启动时输出关键配置摘要
- 敏感配置变更写入审计
- 关键开关默认关闭，需显式启用

## 12. 威胁到控制映射表

| 威胁 | 主要控制措施 |
| --- | --- |
| 未授权进程连接 IPC | UDS、本地凭证校验、包名和签名白名单 |
| 重放高危请求 | `request_id` 去重、挑战 TTL、会话过期 |
| 任意命令执行 | Shell 模板白名单、结构化参数、超时限制 |
| 任意文件读取 | 路径白名单、规范化路径校验、读取上限 |
| 截图隐私泄露 | 审计仅记录摘要、缓存 TTL、日志脱敏 |
| 适配失效误操作 | 版本范围、熔断、回退到无障碍或人工确认 |
| 高频请求 DoS | 限流、并发上限、订阅上限、健康检查 |
| 事后无法追踪 | 结构化审计日志、请求与会话 ID |

## 13. 剩余风险

以下风险在当前架构下被视为可接受但不可完全消除：

1. **设备已被更高权限恶意代码接管**
   - 若攻击者已完全控制 Root 或更底层环境，Clawdroid 无法提供强保证。

2. **目标 App 频繁更新**
   - 适配代码始终存在失效窗口期。

3. **自动化误操作**
   - 模型理解误差、OCR 偏差、页面瞬时变化可能导致错误执行。

4. **ROM 差异**
   - 不同厂商实现差异会带来部分不可预测行为。

这些风险必须通过灰度发布、能力降级、审计与版本兼容治理持续控制。

## 14. 安全实现要求

实现阶段必须满足以下要求：

1. **协议实现**
   - 完整遵守 [protocol.md](file:///d:/temp_desktop/Proj/Clawdroid/Docs/protocol.md) 中冻结的字段、状态机和错误码。

2. **认证实现**
   - 不得仅依赖 UID；必须同时实现凭证校验、包名校验、挑战应答和会话控制。
   - 签名校验在开发联调阶段允许配置为空，但发布版和对外测试包必须启用非空 `auth.allowed_signatures`。

3. **执行实现**
   - `inject_tap`、`inject_swipe`、`exec_shell_limited`、`read_file_limited` 必须做参数范围和授权检查。

4. **适配实现**
   - LSPosed 适配代码必须采用包名白名单与版本范围控制，不得默认对系统全局生效。

5. **日志实现**
   - 高危动作必须有审计记录，审计日志不得记录明文敏感图像内容。

6. **清理实现**
   - 截图缓存、临时文件、失效会话和旧日志必须定期清理。

## 15. 安全测试要求

进入 MVP 联调前，至少完成以下测试：

1. **未授权连接测试**
   - 模拟其他 App 或本地进程直接连接 UDS，验证认证失败路径。

2. **重放测试**
   - 重复发送同一 `request_id` 和旧会话令牌，验证重放防护。

3. **参数越界测试**
   - 发送越界坐标、超大文件读取、非法事件名和超长负载。

4. **Shell 白名单测试**
   - 验证不在模板白名单中的命令全部被拒绝。

5. **文件路径穿越测试**
   - 验证相对路径、软链接和白名单外路径均无法读取。

6. **适配熔断测试**
   - 连续制造适配失败，验证熔断与回退策略。

7. **日志完整性测试**
   - 验证高危动作均产生审计记录，失败动作也可追踪。

## 16. 上线前检查清单

上线或发布测试版前必须逐项确认：

- 是否关闭了 ClawRuntime 的 TCP 监听或调试接口
- 是否启用了对端凭证校验、签名校验、挑战应答和会话 TTL
- 发布包是否已配置非空 `auth.allowed_signatures`
- 是否完成动作级能力映射和参数范围校验
- 是否对 Shell 模板、文件白名单和截图缓存做了限制
- 是否为所有高危动作接入了审计日志
- 是否为 LSPosed 适配模块配置了白名单与熔断
- 是否完成了最小兼容矩阵上的安全回归测试

## 17. 文档维护规则

本文档属于安全基线文档，维护规则如下：
