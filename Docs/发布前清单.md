## 3. 构建步骤

### 3.1 ClawApp

`ClawApp` 构建前需先确认 Runtime 共享密钥来源已经准备好，满足以下任一条件即可：

- 设置环境变量 `CLAWDROID_RUNTIME_SHARED_SECRET`
- 在仓库根 `local.properties` 中设置 `clawdroid.runtime.sharedSecret=<64位随机串>`
- 可从 [local.properties.example](file:///d:/temp_desktop/Proj/Clawdroid/local.properties.example) 复制初始化
- 发布包场景建议同时设置 `clawdroid.runtime.allowedSignatures=sha256:...`

建议至少执行一次 Kotlin 编译校验：

```bash
cd ClawApp
.\gradlew.bat :app:compileDebugKotlin
```

如需生成 APK，可在 Android Studio 中直接构建，或继续执行：

```bash
cd ClawApp
.\gradlew.bat :app:assembleDebug
```

### 3.2 ClawRuntime

建议先同步共享密钥，再进行 Runtime 与模块打包：

```powershell
cd ClawRuntime
.\scripts\sync-shared-secret.ps1
```

先构建 Android 运行时二进制：

```powershell
cd ClawRuntime
.\scripts\build-runtime.ps1
```

再打包 Magisk ZIP：

```powershell
cd ClawRuntime
.\scripts\build-magisk.ps1
```

说明：

- `build-magisk.ps1` 会自动调用 `sync-shared-secret.ps1`
- 当前默认将仓库根 `local.properties` 中的 `clawdroid.runtime.sharedSecret` 同步并生成 `magisk/config/runtime.generated.yaml`
- 打包阶段会把本地生成配置映射为 ZIP 内的 `config/runtime.yaml`
- 仓内公开模板 `magisk/config/runtime.yaml` 不应保留真实密钥
- 若是发布或对外测试包，建议使用 `.\scripts\build-magisk.ps1 -RequireAllowedSignatures`

默认输出：

- 运行时二进制：`ClawRuntime/magisk/bin/clawdroid-runtime`
- Magisk ZIP：`ClawRuntime/dist/ClawRuntime-magisk.zip`
- 本地生成配置：`ClawRuntime/magisk/config/runtime.generated.yaml`

## 6. 发布关注点

发布前建议重点留意以下事项：

- `ClawApp` 首次安装后，仍需要用户完成系统授权：
  - 通知权限
  - 无障碍服务
  - 修改系统设置
  - 全部文件访问
- 发布版 `ClawRuntime` 不应保留空的 `auth.allowed_signatures`
  - 当前运行时允许开发联调时留空
  - 发布包必须填入明确的签名摘要白名单并做真机验收
- 仓库公开状态下不应提交真实 `auth.shared_secret`
  - 真实密钥只应存在于本地 `local.properties` 或环境变量
  - 打包时由脚本生成 `runtime.generated.yaml`
- 发布包的 `auth.allowed_signatures` 不应继续依赖人工手工编辑
  - 推荐通过 `clawdroid.runtime.allowedSignatures`
  - 或环境变量 `CLAWDROID_RUNTIME_ALLOWED_SIGNATURES`
  - 并使用 `-RequireAllowedSignatures` 强制校验
- 若设备支持 Root，概览页已提供：
  - Root 一键授权
  - Root `chmod`
  - Root `chown`
- `ClawRuntime` 现在不再自动迁移旧模块输入
  - 旧设备若仍保留历史模块，需要先手动备份配置，再手动卸载旧模块
- “无障碍感知快照”已做长度收口
  - 仍建议真机上观察高频页面切换时的刷新体验
- 截图预览依赖 `file_bridge_enabled`
  - 若设备上是旧配置文件，需要确认 `config/runtime.yaml` 中该项为 `true`
- 模块 WebUI 当前是本地静态诊断快照
  - 用于展示状态、自检摘要与日志 tail，不作为远程控制台使用

## 7. 回滚思路
