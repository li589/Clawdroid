模块摘要

当前模块 ID 为 clawruntime，运行时入口为 bin/clawdroid-runtime，默认配置文件为 config/runtime.yaml。

WebUI 通过读取同目录下的 status.json、verify.json、verify.txt 和日志 tail 文件显示模块实时信息。

安装后检查

  • 确认 /data/adb/modules/clawruntime/bin/clawdroid-runtime 存在且可执行
  • 确认 /data/adb/modules/clawruntime/config/runtime.yaml 已落盘且内容正确
  • 如果刚更新模块，请先重启设备，再回到 App 执行 Ping / Runtime Probe
  • 如果截图成功但无法预览，优先检查 capability.file_bridge_enabled 是否为 true

常用命令

建议通过 adb shell su -c '...' 执行。

sh /data/adb/modules/clawruntime/verify.sh cat /data/adb/modules/clawruntime/module.prop ls -l /data/adb/modules/clawruntime/bin tail -n 80 /data/local/tmp/clawdroid/logs/runtime-service.log tail -n 80 /data/local/tmp/clawdroid/audit/runtime-audit.log

App 联调建议

  • 先在概览页执行 Ping、Version、Health、Capabilities
  • 再执行 Runtime Probe，确认会话握手与能力同步链路正常
  • 截图链路建议连续执行：Capture Screen -> 读取并预览最近截图
  • 聊天页已支持自然语言快捷动作，如 获取能力、截图并预览、点击 540 1200

故障提示

如果 verify.sh 通过但 App 侧仍无法联通，优先检查设备上的 Root、无障碍、LSPosed 注入状态，以及 Runtime Socket 是否被安全策略拦截。

如果模块曾安装过历史版本，请确认旧模块已手动移除，避免配置目录或日志目录被旧状态干扰。
